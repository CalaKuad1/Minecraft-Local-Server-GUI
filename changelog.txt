# Minecraft Server GUI - Changelog

## Version 1.2.0 - 2026-04-30 UI REDESIGN & NEOFORGE

### UI REDESIGN:
- **Pixel art icon system** — Custom 16x16 pixel icons replacing generic icons throughout the entire UI
- **Visual effects** — Mouse spotlight tracking, noise grain overlay, magnetic button hover effects, abstract animated backgrounds
- **Server Library** — Completely redesigned server selector with grid view, search bar, recently opened section, and engine type icons
- **Refined dark theme** — Deeper blacks, improved contrast, smoother glassmorphism panels
- **Error Boundary** — Graceful error handling across the UI with fallback components

### NEOFORGE SUPPORT:
- Full support for **NeoForge** as a server engine
- NeoForge appears in Setup Wizard, Mods browser, and server cards with its own dedicated icon
- Java version mapping correctly handles NeoForge/NeoMinecraft 26.x+ to Java 25

### MODPACKS:
- New **Modpacks** tab in the Mods section — browse and install complete modpacks directly from Modrinth
- Full filtering support: loader, version, category, and sort order

### MULTI-LANGUAGE (i18n):
- Interface available in **English, Spanish, French, and Russian**
- Language selector in the new App Settings panel
- Centralized LanguageContext for easy extension

### APP SETTINGS:
- New dedicated **Settings panel** with language, theme, notifications, auto-start, minimize-to-tray, and update check options

### IMPROVEMENTS:
- **WebSocket Context** — Centralized real-time communication layer
- **Engine icons** — Each server type (Vanilla, Paper, Spigot, Forge, NeoForge, Fabric) has its own visual icon in cards and selectors
- **Improved Mods browser** — Filter by loader (Fabric, Forge, NeoForge, Quilt), version, category, and sort order
- **GitHub Actions** — Added build verification step for both Windows and Linux before publishing releases

## Version 1.1.9 - 2026-04-08 LINUX STABILITY & SAFETY UPDATE

### CRITICAL BUG FIXES:
- **Fixed read-only filesystem crash on Linux** — All data goes to `~/.minecraft_server_gui/` on Linux
- **Fixed RAM leak on Linux** — Memory-safe log structures (`deque` instead of `list.pop(0)`)
- **Fixed unkillable app on ALL platforms** — App only stays in background when servers are actively running

### SAFETY IMPROVEMENTS:
- **RAM allocation cap** — Slider limited to 80% of system RAM
- **RAM warning system** — Visual warnings when assigning >50% or >75% of system RAM
- **System info endpoint** — New `/system/info` API returns total RAM

### HOTFIX 1.1.9.1 (Included in v1.1.9):
- Fixed backend restart loop on Windows
- Improved Java version mapping for Java 25
- Fixed Startup Scripts (run.bat/run.sh) environment injection
- Fixed Portable Executable Stability
- Robust Forge version handling

### IMPROVEMENTS:
- Smart close behavior
- Tray icon fixes for Linux
- Parent watchdog fix
- Memory-optimized log storage
- New endpoints — `/servers/running`, `/system/info`
- Multi-server improvements
